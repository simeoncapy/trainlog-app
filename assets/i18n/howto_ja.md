# 一般

アプリケーションは、**マップ**、**トリップリスト**、**リーダーボード**、**統計**の4つの主要な部分に分かれています。これらは下部のナビゲーションバーからアクセスできます。その他の機能はメインメニュー内にあります。左上のフローティングバーガーアイコン :icon(menu): で開くことができます。

# マップ

マップにはすべての旅行経路が表示されます。色は交通モードによって異なります（設定でカラーパレットを変更できます）。予定されている旅行はハッチング（斜線）で表示され、進行中の旅行は赤色で表示されます。特定の旅行を表示するためのさまざまなフィルターが用意されており、右下のボタンをクリックするだけで利用できます。

さらに、マップを管理するために3つのツールが使用可能です。1つ目 :icon(my_location): は、マップを現在の位置に再配置します（Trainlogへのアクセスを許可している場合）。ダブルタップするとズーム値がリセットされます。2番目のオプションは、移動中にマップを自分の位置に自動で合わせます。:sym(frame_person): をタップして有効にし、:sym(frame_person_off): をタップして無効にします。最後 :icon(explore): は、マップを北向きに再調整します。

任意の経路をクリックして、旅行の概要シートを表示できます。そこから、旅行の共有、編集、複製、削除が可能です。共有オプションについては、設定で有効にする必要があります。旅行はリンクとして共有されます。

# トリップリスト

過去および将来のすべての旅行（上部のトグルセレクターを使用）をページ分けされたビューで表示します。テーブルを水平方向にスクロールして、旅行に関する詳細情報を表示できます。すべての詳細を表示するには、行をクリックしてボトムシートを表示します。テーブルを下にドラッグすると、サーバーと同期してトリップリストが更新されます。

## 旅行の追加 - 基本

まず、駅の検索に影響するため、使用した車両のタイプを選択します。次に、名前を入力して駅を選択できます。ミニマップを使用して、選択内容が正しいかどうかを確認できます。これは、大きな駅が異なるエンティティ（文字の接尾辞付き）に分割されている場合に特に便利です。:icon(fullscreen): をクリックしてミニマップを拡大できます。

駅が存在しない場合は、手動で駅を作成できます。これを行うには、:sym(globe_location_pin): をクリックしてモードを変更します。次に、駅の名前とその座標を入力します。座標がわからない場合は、ミニマップを拡大した後にピンを移動することもできます。

フォームの最後で、旅行の1つまたは複数の運行会社を選択できます。既存のものが表示されます。運行会社が存在しない場合は、Enterキーまたはカンマでフィールドを確定して、ロゴなしの運行会社を作成できます。プロジェクトのDiscord（「Trainlog」タブを確認）でロゴの追加をリクエストできます。

## 旅行の追加 - 日付

画面上部で選択可能な3つの日付モードが用意されています：

- **正確**: 正確な出発・到着の日時を入力します。両方のフィールドに日付選択と時刻選択が含まれています。タイムゾーンは駅の座標から自動的に推測され、各時刻フィールドの下に表示されます。遅延セクションを展開して、実際の遅延を記録することもできます。遅延を分単位で入力するか、実際の出発/到着時刻を直接設定します。
- **日付**: 旅行日のみ（時刻なし）入力します。オプションで旅行時間を時間と分で指定できます。
- **不明**: 正確な日付がわからない場合に使用します。旅行が過去か未来かを選択し、オプションでおおよその時間を入力します。

## 旅行の追加 - 詳細

このページのすべてのフィールドはオプションです。

**路線**番号または名前、**車両**（車両モデル）、車両の**登録**番号、**座席**番号、およびフリーテキストの**メモ**を記入できます。

**チケット**セクションでは、チケットの価格（通貨セレクター付き）と購入日を記録できます。

**エネルギー**セクションでは、車両の牽引タイプ（自動、電気、または燃料）を指定できます。

**公開設定**セクションでは、この旅行を誰が見ることができるかを制御します。非公開（自分のみ）、友達のみ、または公開です。

## 旅行の追加 - 経路

経路ステップには、出発駅と到着駅の間のルートを自動的に計算するインタラクティブマップが表示されます。距離と推定時間が上部に表示されます。鉄道、地下鉄、トラムの旅行では、**新しいルーター**オプションを切り替えて代替のルーティングエンジンを使用できます。詳細についてはヘルプアイコンをタップしてください。経路に満足したら、**確定**を押して旅行を保存します。また、**旅行を続ける**を押すと、保存してすぐに現在の到着駅を新しい出発駅として新しい旅行を開始できます。

# ランキング

ランキングページには、各車両カテゴリーのすべてのTrainlogユーザーのリーダーボードが表示されます。

# 統計

統計ページでは、チャートやテーブルを通じて旅行データを探索できます。上部のフィルターパネルを使用してビューをカスタマイズします。タップして展開または折りたたみます。

利用可能なフィルターは次のとおりです：
- **車両**: 分析する輸送のタイプ。
- **年**: 特定の年でフィルターするか、完全な履歴を表示するために「すべての年」を選択します。（グラフの種類が「年」に設定されている場合は無効になります。）
- **グラフの種類**: データを何で分類するかを選択します — 運行会社、国、年、車両、または旅程。
- **単位**: 指標を選択します — 旅行回数、距離、時間、またはCO2。

右上隅のセレクターから3つのチャートタイプを利用できます：
- **棒グラフ**: 上位10項目を棒グラフとして表示します。トグルにより、水平方向と垂直方向の向きを切り替えることができます。
- **円グラフ**: 上位10項目を円グラフとして表示します。
- **テーブル**: データセット全体をソート可能なテーブルとして表示し、過去と将来の旅行用に別々の列を表示します。トグルにより、合計値によるソートとアルファベット順を切り替えることができます。

すべてのチャートタイプにおいて、過去と将来の旅行が並んで表示されます。

# Geolog (スマートプリレコーダー)

Geologはメインメニューからアクセスできるスマートプリレコーダーです。出発と到着の瞬間に現在の位置を記録し、それら2つの記録を使用して自動的に旅行を作成できます。

**記録**ボタンをタップしてgeologを保存します。アプリは現在の座標とタイムスタンプを取得し、設定された半径内（設定で調整可能）の最寄りの駅を検索します。近くに複数の駅が見つかった場合は、正しい駅を選択できるように選択ダイアログが表示されます。駅が見つからない場合、geologは不明な場所として保存されます。すべてのデータはデバイスのローカルにのみ保存されます。

旅行を作成するには、リストから正確に2つのgeologを選択します。最初の1つは出発（**発**とマーク）、2番目は到着（**着**とマーク）として処理されます。次に、**旅行を作成**をタップして、保存された場所とタイムスタンプが事前入力された旅行追加フォームを開きます。旅行が保存されると、2つのgeologは自動的に削除されます。

個別のgeologを選択して**選択を削除**をタップするか、**すべて削除**で一度にすべてを削除できます。ソート順（新しい順/古い順）はソートボタンで切り替えることができます。
